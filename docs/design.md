# 设计说明

## 目标
把新闻链接转换成一条可直接发布的评论。

## 当前输入输出
- 输入：一个新闻 URL
- 输出：一条主推评论文本

命令行不再输出结构化报告，也不再支持样例文件、本地文本或 JSON 输入。

## 核心流程
1. `ingestion`
   抓取 URL，对正文做轻量提取。
2. `understanding`
   提炼核心观点、争议点、笑点和讨论切口。
3. `reddit_retrieval`
   通过搜索引擎查找相关 Reddit 帖子，再抓取帖子可读内容提取评论；失败时回退本地样例。
4. `pattern_derivation`
   从参考评论里提炼互动模式。
5. `generation`
   生成多条候选评论。
6. `ranking`
   选出主推评论。

## 关键取舍
- 简单优先：CLI 只保留 `--url`
- 输出收缩：只返回评论正文
- 在线优先：先尝试在线检索 Reddit 参考评论
- 稳定性兜底：在线失败时回退本地 Reddit 样例

## 已知限制
- 对复杂动态网页，正文提取不一定稳定
- 无凭证 Reddit 抓取依赖搜索结果和可读代理
- 当前仍然是单条评论生成器，不包含自动发布
